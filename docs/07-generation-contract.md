# 07 · 生成契约（铁律）

> 这是单一事实源。每次生成社媒卡片**开工前必读本文件**；出现偏差时，回到这里逐条核对再重做。
> guizang 提供的是方法论（skill），本文件是我们把它落到 md2red 的硬规则——只取方法，不照搬其命名 / 字体 / 主题。

---

## A. 生成前必过流程（Preflight Checklist）

每生成一篇，按顺序走完 1–9，任何一步不得跳过：

1. **读规则** — 读本文件 + `docs/00-system.md`，确认最新约束。
2. **抓原文 + 阶梯提取** — 加载原文，提取：核心主张 / 给读者的承诺 / 段落地图（每个小标题 = 一个候选正文页）/ 钩子金句 / 关键数字与来源。逐条标注来源**用于核实**（无可靠来源的事实不得进卡）；但**来源只用于核实、不在卡面展示**（见 E「卡面不展示来源」）。
3. **定页数** — 字数→页数：600–1000 字 → 5–7 页；1000–1800 → 7–9；超长 → 8–10。封面 + 正文（每条一页）+ 尾页。
4. **判形态 → 选正文版式** — 走路由表（`docs/03-content-mapping.md`），据语义为这篇选最贴合的正文 recipe（见 E），不固定死模板。
5. **写标题** — 走 C 的四步；数字不进标题。
6. **填充** — 按所选 recipe 填真字段，密度 ≥75%；正文页按**顶锚定等高节奏**排版（见 E）：标题带顶锚定 + 设 `min-height` 跨页等高，焦点带 `min/max` 吸收内容多寡，剩余留白收到底部有界画框；带间按语义层级（标题↔导语最紧 `--s2`、导语→焦点 `--s5`、焦点→操作最大 `--s6`）；密度靠填满焦点带的真字段达成，**禁 spacer / flex:1 压底撑版，也不留无理由空洞**。
7. **尾页** — 按 D 的 M07 规则升华。
8. **去 AI 味（出卡前必过）** — 文案定稿后，按 `docs/09-humanizer-pass.md` 过一遍正文文案：清 AI 词黑名单 / `-ing` 肤浅分析（「…，彰显 / 体现了…」）/ 否定式排比 + 金句 / 三段式硬凑 / 宣传升华腔 / 通用积极结论 / 填充词 / 销售性破折号；5 维评分 ≥40/50 才放行。**只清正文文案；组件骨架（`.num`/`.idx`/`.how` 标签/`ledger` label/`finding` 三锡点/标题与分隔的破折号）豁免；被 G 冻结的部分不动。**
9. **自检** — 走 F 清单，任一不过则回到对应步骤重做。

## B. 页码 / 进度条规则（锁死）

- 正文页右上角的进度指示**只能用 card-base 定义的 `.dots` 组件**，不得每次手写 `01 / 06 · 晨光` 这类自由文本。
- 实现：`.header > .h-right` 内放 `.dots`，内含 N 个 `<i>`（N = 正文页总数），当前页对应的那个加 `.on`。
- guizang 硬规则：**一张海报只能有一个 issue / 页码类元数据，不能上下都放**。页码统一放 header h-right；封面 / 尾页可不放。
- 不允许“生成一次变一个样子”——dots 的数量、形态、位置由 card-base 决定，内容层不得改写。进度指示**不带「01/06」数字**（数字拖累阅读焦点，只用点/长条表位置）。

## C. 标题提练规则（锁死）

来源：guizang `title-shortener.md`。**禁止读一段就自由发挥扩写。** 四步：

1. **取忠实小标题** — 优先用原文已有的 subhead（如 `1. Do the Sun Dance`），不绕开它另起炉灶。
2. **直译** — 译成中文（Do the Sun Dance → 晒晨光 / 来跳支晨光舞）。
3. **压成 动词 + 对象，4–12 字** — 去掉修饰语，只留核心动作 + 核心对象（→「去晒晨光」）。
4. **副标题仅用于消歧**，正常不加。

**硬禁项：**

- **数字不进标题**（除非数字本身就是这条的“新闻”且为原文真实数据 = Pattern D；即便如此也不得与数据框矛盾）。`花20分钟` 这种把数据塞进标题、又和下方 `20–30 分钟` 打架，属硬错。
- 不得“想说什么说什么”——标题的每个语义都要能在原文 / 数据框找到对应，**不得炫技扩写、加戏**。
- 不用「如何 / 怎样 / 为什么」做开头；不出现 3 行标题；内容是中文时不用纯英文标题。

## D. 尾页升华规则（M07，锁死）

来源：guizang `layout-recipes.md` M07 + `components.md`。

- **基于什么升华**：只允许复述正文**已出现**的要点 / 后果 / 来源，**禁止引入任何新事实、新数字、新主张**。
- **升华到什么程度**：
  - 一行收束标题（≤2 行），重新框定全篇主旨（不是招新论点）。**字号不超过正文 h2 量级，别做成超大标题。**
  - **4–6 条 grounded recap**，每条 = 要点标题 + 一句后果 / 原因 / 例子（对应正文某页）。
  - **一个**收束块：金句 / CTA / 旁注，三选一（一句行动话即可）。
- **署名只在每页页脚 `.brandfoot` 出现一次**，尾页收束块**不再重复署名块**（否则一页两处署名）。
- **硬禁项**：不得只放 3 条短账本行（= guizang 明确点名的 failure mode）；不得用销售腔 / 夸大。收束标题**不用否定式排比 / 金句腔**（见 09）。
- 署名落款统一 `@津玉不良言`（仅 `.brandfoot`）。

## E. 正文版式选择 + 当前默认 recipe

- **不固定死模板**：每篇按语义在路由表里选最贴合的正文 recipe（M01–M16 / Swiss S01–S12）。
- **混用 vs 一致（关键边界）**：「允许混用结构」只对**异质内容**成立——散文 / 多段落 / 角色各异的长文（封面→场景→误区→证据→金句→收尾），靠换版式制造节奏。**编号清单体（对等条目清单，「N 个 X」且各条地位平等）恰好相反：全篇正文页保持同一配方家族**，因为各条是并列「同类项」，版式一致才传达「这是一组」。往清单流里插 `finding`/`step`/`matrix` 等异形页，会让个别条目「格格不入」，破坏组的整体感。
  - 清单内**唯一允许的变化** = 同一 `data` 家族里「该条有无真实数字」：有数字走 `data` + `.stat`，无数字走 `data` 文字主导（`.pull` + `.how`，**不带 `.num`**）。骨架不变，不算换配方。
  - 反例：HE-0001《6 个晨间仪式》把「提早起床」「冥想塑身」做成 `finding`，与其余 `data` 条目割裂——错。六条应统一走 `data` File Card。
  - 何时才该整篇换形态：当这篇本身不是对等清单，而是单点强论（整篇走 `finding`）/ 步骤流程（`step`）/ 对比测评（`matrix`）——那是形态不同，不是「在清单里插异形页」。
- **卡面不展示来源出处（锁死）**：正文卡片**一律不渲染** `.src` / `.src-line`（组件里这两个类 = 「期刊名 / 研究出处」那行）。来源只在 §A 第 2 步提取时记录、用于**核实数据真伪**，不印在图上——数据可信度靠「机制说明 + 真实数字」承载，不靠在卡上挂期刊名。（`finding` 的三锡点中，第三锡点改用说明行 `.cap`，不用来源行。）

### 默认：数字主导 File Card（S03 Data/File Card + Pattern D）

适用于“每条都有一个真实研究数字 + 机制 + 做法”的清单型干货（如 Eat This 晨间仪式）。**正文页走顶锚定等高节奏（N11）**：

```
┌ 标题带（顶锚定，紧接页眉，设 min-height 跨页等高）──
│ [序号 chip · 标题]      ← 同一行；序号小，标题 = C 提练出的「动词 + 对象」
│ 一句导语（承上启下）
├ 焦点带（填色面板 card-2 + 左 8px 锚点边；min/max 吸收内容）──
│ 大数字 + 单位<em>   ｜ 无数字则用一句主张（.pull/.bigtext，同一面板）
│ 机制（为什么有效）        ← 来源不上卡，只在提取阶段核实
├ 操作带（描边条 + 锚点圆图标）──────
│ 怎么做：原文真实可执行 tip
└ 标题带顶锚等高；焦点带 min/max；剩余留白收到底部有界画框
   带间：标题↔导语 --s2、导语→焦点 --s5、焦点→操作 --s6
```

- **节奏是这个 recipe 的一部分**：标题顶锚等高 + 底部有界留白，是为了修掉「页面很素 / 一点间距设计都没有」：内容不再头重脚轻、不再吊一大片死白，且跨页标题对齐。
- **焦点带 = 一个填色面板**（`--c-card-2` 底 + 左侧 8px 锚点边），这是 N15 的正式松动：让 B/D 不再「素」，但仍全套单锚点色、无装饰图。每页至多一个这样的面板（= N12 一个 `.stat`）。
- **解决留白的唯一合法手段 = 加真字段**（机制 / 做法都是原文本来就有的），不是拉伸、不是 spacer 压底。密度仍须 ≥75%。
- **一页一焦点**：序号必须小（chip），避免和大数字抢焦点（修掉旧版“大 01 + 大数字”双焦点问题）。序号后不加无意义小字。
- **禁把异质单位（分钟 / 倍 / 周）做成同一张图表**（KPI 塔 / 条形 / 矩阵），那等于伪造可比性，违反 guizang「禁造假数据」。

## F. 自检清单（出图前必过）

- [ ] **节奏**：正文页走顶锚定等高（标题顶锚→焦点→操作），标题带跨页等高，带间按语义层级（标题↔导语 `--s2`、导语→焦点 `--s5`、焦点→操作 `--s6`）；填满焦点带、无 `flex:1` / spacer 压底撑版，剩余留白收到底部有界画框（底部画框留白是有意的，不算欠填）。
- [ ] **一页一焦点**：每页只有一个明显的视觉焦点；至多一个填色面板（= 一个 `.stat`）。
- [ ] **色彩层级**：60-30-10，锚点色只落在焦点（chip / 面板左边 / 单位 em / 操作图标），未泛到正文与元信息。
- [ ] **页码**：正文页 header h-right 用 `.dots`，数量 = 正文页数，不带「01/06」数字，且全页只有一处元数据。
- [ ] **标题**：每个标题来自忠实 subhead，动词 + 对象 4–12 字，无数字，**不与数据框矛盾**；序号与标题同行。
- [ ] **来源**：每个数字 / 事实在提取阶段可溯源（核实用，无杜撞）；**卡面未渲染任何来源行**（`.src` / `.src-line` 不出现在成品卡上）。
- [ ] **去 AI 味**：正文文案已过 `09-humanizer-pass`——无 AI 词黑名单命中、无否定式排比 / 金句腔、无 `-ing` 肤浅分析、无销售性破折号；5 维评分 ≥40/50。组件骨架与冻结部分不在清理范围。
- [ ] **尾页**：M07，4–6 条 recap + 1 收束块，收束标题不超大，无新事实，无重复署名块，无金句腔。
- [ ] **结构一致 / 多样（看形态）**：编号清单体等**对等条目清单**——全篇正文用**同一配方家族**（仅 `data/stat` ↔ `data` 文字主导 之差），禁插异形页；异质长文——用 ≥5 种形状、避免全篇同一版式。
- [ ] **冻结校验（见 G）**：本次只改了用户点名的范围，已冻结部分与上一版 approved commit 逐字一致。

## G. 锁定 / 冻结（approved 部分不回退）

- 用户对某一部分明确说“没问题 / 就这样 / 别动”后，该部分即**冻结**。
- 之后任何“覆盖 / 重做”，**只能改用户当次点名的范围**；范围之外的一切（尤其已冻结部分），必须**从上一版 approved commit 逐字保留**，严禁凭记忆重写或顺手“优化”。
- 覆盖前先在脑子里 diff：这次要动哪几页 / 哪几块？其余原样搬运。
- 去 AI 味闸（09）**不得越过冻结**：冻结部分即便有 AI 腔也不动。
- guizang 没有这条（它是 skill，无“逐次审批”语境）；这是 md2red 对话式生成必须补的规则。
