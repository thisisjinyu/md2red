# 09 · 出图前 QA 全清单（QA Checklist）

> 对齐 guizang `references/qa-checklist.md`，裁到 md2red 的「黑白文本健康卡」范围。
> 每套图出图前逐条过；任一不过 = 改，不是「顺眼就行」。机器可查的部分见 `scripts/validate-deck.mjs`（规则号 R1–R7 对应见末尾）。
> 这是 `06-rules.md` C 清单的**超集**：06 管硬规则自检，本文件管「交付前总检 + 渲染态检查」。

---

## A. 尺寸 Dimensions

- [ ] 每张卡画布 `1080×1440`（3:4）。
- [ ] 文件名稳定、按页序排列（`decks/<slug>/` 下）。
- [ ] （若做了主图封面）尺寸符合 `10-image-cover.md` 约定。

## B. 文字 Text

- [ ] 无文字溢出 / 贴边。
- [ ] 无内容压到 `.brandfoot`。若正文长出来撞页脚：页脚应是 `margin-top:auto`（**禁 `position:absolute`**），见 `06-rules.md` 反例 C。
- [ ] 封面主标在手机缩略图尺寸下仍可读（缩到 360px 宽自查）。
- [ ] 正文字号不小于需要——**最小 22px**，塞不下删内容、不缩字号（N8）。
- [ ] 中文换行是刻意的（`<br>` 落在语义断点，不是随机折行）。
- [ ] 专有名词 / 英文 / 数字拼写正确。
- [ ] 文案对应原文，**无杜撰事实**；未核实标「示例数据 · 待核」/「来源 · 待核」。

## C. 版式 Layout

- [ ] 每页一个明确焦点（`data` 页序号要小，别和大数字抢焦点）。
- [ ] 全篇不是所有页同一结构（一套 ≥5 种形状，见 `08-recipe-catalog.md` D）。
- [ ] 下方空白要么刻意（`finding`），要么补满。
- [ ] 元素对齐 88px 边距与 8px 节奏。

### 4-band 密度检查（3:4，渲染后做）

把渲染图竖向均分 4 条带（各 360px）。每条带判：

- **填充** —— 有文字/图/数据/线。
- **正当留白** —— 有理由的空：`finding` 单句氛围、首尾边距（各 ≤8%）。
- **欠填** —— 无理由的空。**任一欠填带 >15% 画布高（>216px）= 不过。**

判过条件：① 填充 + 正当留白 ≥100%；② 填充带覆盖 ≥75%（≥1080/1440px）；③ 不存在相邻两条带都「正当留白」（否则中部 >25% 空洞）。

不过怎么办：**别缩画布、别加装饰块**。要么扩 copy（多 ledger 条 / 长段落 / 来源行 / 边栏），要么换配方（`close→finding`、薄表`→ledger`）。

## D. 风格 / 身份 Style

- [ ] 一套图只用**一个主题**（B/C/D 三选一，不混搭，N13）。
- [ ] **md2red 身份测试**逐条过（见 `06-rules.md` E）：单锚点色、大字细体（封面主标 250）、无圆角无阴影（C 发光除外）、无 emoji、无随机 SVG 装饰块。
- [ ] 光晕 `.aura` / 幽灵数字 `.ghost` / 渐变描字只在 C 主题出现。
- [ ] 无嵌套卡片、无多余圆角。

## E. 图片（仅当用到主图封面）

> 仅在「文字压在照片上」时跑（主图封面）。完整规则见 `10-image-cover.md`。

- [ ] 照片过「静区测试」（≥30% 画布低细节带）+「光线测试」（氛围光，非高饱和正午）。
- [ ] 先试**无蒙版**构图；只有缩略图对比度不过才加局部、图调色、峰值 α≤0.30 的蒙版。
- [ ] 无整屏竖向黑色渐变（game key-art 做法，禁）。
- [ ] 主体地图写成 HTML 注释贴在主图块旁。
- [ ] 没有 ≥72px 大标压在脸 / 手 / 关键主体上。
- [ ] `object-position` 按主体位置表选，不是默认。
- [ ] 缩略图测试通过（缩到 360px 宽，标题仍清晰）。

## F. 最终交付 Final

- [ ] 给出输出路径 / deck 链接。
- [ ] 需要时内联展示主封面。
- [ ] 说明做了哪些核对。
- [ ] 说明局限（如低分辨率素材、数据待核）。
- [ ] **冻结校验**：本次只改了用户点名范围，已冻结部分与上一版 approved commit 逐字一致（`07` G）。
