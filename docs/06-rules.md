# 06 · 硬规则与反模式（Rules）

> 生成完一套卡片，照这份逐条自检。任何一条不过 = 重做，不是「顺眼就行」。
> 数值/控件细节见 `05-components.md`；拼卡见 `04-layouts.md`；配色见 `02-palettes.md`；配方见 `08-recipe-catalog.md`；去 AI 味见 `09-humanizer-pass.md`。

---

## A. 不可谈判（Non-Negotiables）

### A1 尺寸与结构
- **N1** 卡片永远 `1080×1440`（3:4）。内容用 `flex` 撑满，**禁用 `vw/vh`**。
- **N2** 内边距恒为 `--margin`（88px），由 `.pad` 统一控制。
- **N3** 所有分隔线恒为 `--hair-w`（2px）。
- **N4** 间距只用 `--s1..s8`（8/16/24/32/48/64/96/120），**不写裸 px**。
- **N5** 品牌脚 `.brandfoot` 靠 `margin-top:auto` 钉底，**禁用 `position:absolute`**。

### A2 字体与字重
- **N6** 封面主标 `.title` 字重恒为 `250`，三主题不变。
- **N7** 大数字（`.index` / `.stat .num`）走 `--big-weight`（B/D=250、C=600），**不单独写死字重**。
- **N8** 文字最小 `22px`（`.src`/`.h-right`）。塞不下→**删内容，不缩字号**。
- **N9** 封面主标里首个数字包 `<span class="d">`（等宽、放大、基线微沉）。等宽栈仅此处用。

### A3 密度与节奏
- **N10** 每张卡内容填满 **≥ 75%** 画布高（顶锚定后留在底部的有界画框留白是有意的，不计入欠填）。
- **N11**（修订·顶锚定等高）正文页走**顶锚定等高节奏**，不再整体垂直居中：
  - **标题带顶锚定**：紧接页眉下方固定起始（不浮在中间）；标题带设 `min-height`，使 1 行 / 2 行标题**占位等高 → 跨页标题基线对齐**。
  - **带间语义层级（不等距）**：标题↔导语最紧 `--s2`（16）｜导语→焦点 `--s5`（48）｜**焦点→操作最大** `--s6`（64，可加发丝线收口）。页眉→标题沿用页眉自带的 `margin-bottom:--s5`，不另定。
  - **焦点带 `min/max` 吸收内容多寡**，不靠拉伸空白。
  - **剩余留白收到底部**，作为**有界画框留白**（非「内容浮在中间、上下对称」）：上紧下松但相对，主区填满 ≥75%。
  - 禁 `.spacer`/`flex:1` 压底撑版；密度靠**填满焦点带的真字段**达成。
  > 废止旧 N11「三带整体垂直居中（`.body{justify-content:center}`）」——居中会让标题位置随内容浮动、跨页不齐。改为顶锚定 + 等高 + 底部有界留白。
- **N12** 一张正文卡**最多一个 `.stat`**（即一个焦点面板）。

### A4 主题纯度
- **N13** 一套图文只用一个主题（B/C/D 三选一），**不混搭**。
- **N14** 光晕 `.aura` / 幽灵数字 `.ghost` / 渐变描字是 **C 主题专属**，B/D 绝不出现。
- **N15**（修订）B/D 仍克制，但**允许一个填色焦点面板**承载页面焦点：用第三表面层 `--c-card-2`（可加左侧 `8px` 锚点色边）框住焦点带——这正是 `.stat` / `.data` 焦点框本来的做法。**上限：每页至多一个填色面板**（= N12）；仍**禁装饰图 / 光效 / 第二种颜色**，全套仍单锚点色。
  > 旧 N15「无填色面板」与现有 `.stat`（本就 `card-2` 底）自相矛盾，且是「页面很素」的根源，已放宽。

### A5 内容诚信
- **N16** 不编造数据。未核实的数据/出处一律标「示例数据 · 待核」。
- **N17** 出卡前正文文案**必过 `09-humanizer-pass`（去 AI 味）**：清 AI 词黑名单 / 否定式排比 + 金句 / `-ing` 肤浅分析 / 宣传升华腔 / 销售性破折号；5 维评分 **≥40/50** 才放行。**只清正文文案，组件骨架与 07 §G 冻结部分豁免。**

### A6 色彩层级（新增）
- **N18** **60-30-10 色彩纪律**：60% 中性底（页/卡面）、30% 次级面（焦点面板 `--c-card-2`、分隔线、元信息）、10% 锚点色。锚点色**只给真正的焦点**：序号 chip / 焦点面板左边线 / 大数字单位 `<em>` / 操作条图标。锚点色稀缺即权威，不得泛滥到正文与元信息（进度点未激活态、序号后小字等走静音色）。

## B. 反模式（看似没错，实则跑偏）

| ❌ 反模式 | ✅ 正确做法 |
| --- | --- |
| 标题太长，把字号调小塞进一行 | 保持 128px，用 `<br>` 换行；实在太长精简文案 |
| 一张正文卡塞两三个数据块 | 一张最多一个 `.stat`，多的另起一张 |
| 内容少就用 `.spacer` 压底 / 或让它浮在中间 | 补真字段（机制/做法）填满焦点带，剩余留白收到底部有界画框（N11）|
| 带间间距随手拍，标题距焦点还没焦点距操作近 | 带间走语义层级：标题↔导语最紧、焦点→操作最大（N11）|
| B/D 因为「克制」就全页大白底、不敢上任何面板，显得很素 | 允许一个 `card-2` 填色面板托住焦点（N15 修订） |
| 用 emoji 当图标/装饰 | 不用 emoji；靠排版、留白、锚点色营造质感 |
| B/D 也加个光晕显得炫 | 光晕只属 C；B/D 靠克制取胜 |
| 锚点色泛到正文/元信息上 | 锚点色只给焦点（10%），其余走中性/静音色（N18） |
| 随手改分隔线粗细/颜色 | 恒 2px + `--c-hair`，不动 |
| 数字单位跟数字同字号 | 单位/百分号用 `<em>`，与大数字拉开层次 |
| 编个好看的数据填上 | 标「示例数据 · 待核」，或留空等真数据 |
| 正文堆「不仅…更…」「不是 X，而是 Y」金句 | 直陈事实，删金句（见 09 去 AI 味）|
| 句末「…，彰显 / 体现了…」加戏 | 删掉 `-ing` 肤浅分析，只留事实（见 09）|
| 正文用「——」制造销售腔停顿 | 改逗号 / 句号；破折号只留给标题与分隔（见 09）|
| 每篇从空白页手搓 | 复制最近的 seed deck 再改（见 00-system「不白手起家」） |

## C. 生成后自检清单

每张卡过一遍：

- [ ] 画布是 `1080×1440`？
- [ ] 正文页走顶锚定等高节奏（标题顶锚→焦点→操作），标题带跨页等高、带间按语义层级、剩余留白在底部有界？
- [ ] 焦点带填满真字段，无大片无理由空白，无 `.spacer`/`flex:1` 撑版？
- [ ] `.brandfoot` 钉在底部且未被内容顶出画外？
- [ ] 每页至多一个填色面板（= 一个 `.stat`），锚点色只落在焦点上？
- [ ] 所有文字 ≥ 22px？间距全是 `--s1..s8`？分隔线全 2px？

整套过一遍：

- [ ] 只用了一个主题？
- [ ] 封面主标字重 250、首个数字包了 `.d`？
- [ ] 光晕/幽灵数字只出现在 C？
- [ ] 每张正文卡 ≤ 1 个 `.stat`？
- [ ] 尾页 `.recap` 条数 = 正文卡数（≤8）？
- [ ] 所有未核数据都标了「示例数据 · 待核」？
- [ ] 正文文案过了 `09-humanizer-pass` 去 AI 味（5 维 ≥40/50）？

> 完整交付清单（含尺寸/文字/版式/4-band/身份测试/图片子集/最终交付）见 `09-qa-checklist.md`。

## E. 身份测试（md2red 单系统版）

> 一套图卡渲出来，问自己这几条；任一为「否」= 不是 md2red 气质，回去改。

- 封面主标是不是 **250 字重的细体大字**，而不是粗黑标题？
- 全套是不是**只有一个主题锚点色**（薄荷蓝 / 酸性霓虹 / 美拉德棕），没混搭？
- 锚点色是不是只落在焦点上（60-30-10），没泛到正文和元信息？
- 大数字是不是走 `--big-weight` 并用 `.d` / `<em>` 拉出层次，而不是和正文一个字号？
- 画面是不是**没有 emoji、没有装饰图**（C 主题光晕除外）？
- 分隔线是不是**统一 2px 发丝线**？
- 正文页是不是**标题顶锚定、跨页等高**，留白集中在**底部有界画框**而非随机空洞（填满焦点带，不靠 `.spacer` 压底）？
- 脚注品牌是不是**钉在底部一次**，没被内容顶穿？
- 正文读起来像不像**人写的**（无 AI 腔、无金句堆砌）？

## F. 渲染态反例库（真机渲出来才暴露的坑）

这些在源码里看不出问题，**渲染成图才暴露**，是 `validate-deck.mjs` 重点抓的：

- **反例 A · 粗大标题**：封面主标被 inline 改成 600+ 字重当「大字报」。
  → 违反 N6/N7；恒 250，靠字号而非字重取势。校验：**R3**。
- **反例 C · 脚注压字**：`.brandfoot` 用了 `position:absolute`，内容一多就和脚注重叠。
  → 违反 N5；改 `margin-top:auto` 钉底。校验：**R2**。
- **反例 D · 主图压脸**：封面用主图时，大标题盖住人物面部 / 关键主体。
  → 选图与蒙版规则见 `docs/10-image-cover.md`（静区选图 + `object-position` + 局部蒙版）。

## G. 机器校验（`scripts/validate-deck.mjs`）

下面规则已落地为脚本（playwright 真机渲染后逐张 `.canvas` 检查）：

```bash
node scripts/validate-deck.mjs decks/he-0001
```

| 规则 | 抓什么 | 对应硬规则 |
| --- | --- | --- |
| **R1** 溢出 | `scrollHeight > clientHeight` | N1 撑满不溢出 |
| **R2** 脚注碰撞 | `.brandfoot` 为 absolute 且被内容顶穿 | N5（反例 C） |
| **R3** 封面字重 | `.cover .title` 字重 ≠ 250 | N6（反例 A） |
| **R4** 最小字号 | 叶文字节点 < 22px | N8 |
| **R5** 4-band 密度 | 下填充 <75% 或单带 >216px 空洞（finding 豁免） | N10 |
| **R6** 标题行数 | `.cover .title` >3 行 / `.close .h` >2 行 | 04-layouts 上限 |
| **R7** 数据页焦点 | 一个 `.data` 内 >1 个 `.stat` | N12 |
| 主题纯度 | `body` 上 theme-* 类数 ≠ 1 | N13 |

> 待调（TODO）：R5 现行密度判定基于旧「居中三带」模型，需改为顶锚定模型——**顶部固定起始、底部有界画框留白**（底部留白约 ≤1 画框单位时视为正当，不计欠填），只对「带内 / 带间」的无理由空洞判 FAIL。另：去 AI 味（N17 / 09）仍是**人工闸**，不在脚本内。

FAIL 退出码 1；WARN 仅提示。手检阶段先照 C 清单 / 09-qa-checklist 走。
