# 01 · 结构与版式系统（配色无关）

这是三套配色共享的骨架。只要遵守本文，任何配色都能保持一致的排版纪律。

## 画布 Canvas

| 项 | 值 | 说明 |
|---|---|---|
| 尺寸 | `1080 × 1440` px | 3:4，小红书 / 公众号通用竖幅 |
| 边距 margin | `88px` | 四边统一外边距 |
| 预览缩放 | `--s: 0.42` | 仅用于画廊预览，不影响导出分辨率 |
| 衡线 hairline | `2px` | 分割线统一粗细 |

## 间距 token Spacing（8px 基准）

所有间距必须是 8 的倍数，保证垂直节奏统一。

| token | 值 | 典型用途 |
|---|---|---|
| `--s1` | 8px | 最小间隙 |
| `--s2` | 16px | 行内元素间距 |
| `--s3` | 24px | 页眉上下内边距 |
| `--s4` | 32px | 标题与正文间距 |
| `--s5` | 48px | 模块间距 |
| `--s6` | 64px | 页眉与正文分区 |
| `--s7` | 96px | 大分区 |
| `--s8` | 120px | 超大留白 |

## 字阶 Type scale

| 角色 | 字号 | 字重 | 用途 |
|---|---|---|---|
| Display | 104px | 250 | 封面主标题 |
| Index | 128px | 250–600 | 正文序号（大背景数字） |
| Num | 150px | 250–600 | 数据块数字 |
| H1 | 60px | 500 | 正文标题 |
| Lead | 36px | 300 | 导读 / 要点 |
| Body | 30px | 400 | 正文 |
| Eyebrow | 25px | 600 | 页眉系列标识（letter-spacing .2em） |
| Caption | 26px | 400 | 数据说明 |
| Source | 22px | 400 | 出处 / 页码 |

> 核心原则：**字号越大，字重越细**。大标题用细体制造留白与高级感；正文保持可读的中等字重。（特例：C 霓虹版为了发光视觉锺点，大字采用 600 重。）

## 统一页眉 Header

每一页（含封面/尾页）顶部都有一条页眉，下接 2px 衡线：

- **左：系列标识**（Eyebrow，如 `HE-0001 · 晨间仪式`），着锚点色。
- **右：进度 / 页码**。封面用「向右滑 →」，正文用进度点（当前页点为拉长锚点色），尾页用「回顾 · RECAP」。

## 卡片类型 Card types

1. **封面 `.cover`** — 页眉 + 主题分类 + 主标题（Display）+ 副文（Lead，上接衡线），底部 brandfoot。
2. **正文 `.body`** — 页眉 + 大序号 Index + H2 标题 + 要点 point + 底部数据块 `.stat`（num / cap / src）+ brandfoot。中间用 `.spacer` 擑开，数据块始终贴底。
3. **尾页 `.end`** — 页眉 + 收尾句 + 回顾列表 `.recap`（逐条序号）+ CTA + brandfoot。

## 品牌署名 Brandfoot

- `@津玉不良言` 固定在**每页右下角**，上接 2px 衡线，`margin-top:auto` 置底。

## 质感原则 Rules

- **无圆角、无阴影**（C 霓虹版可用 drop-shadow 发光作为唯一例外）。
- **单一锚点色**：每套配色只用一个（或一组渐变）主锚点色做信任 / 重点，不滥用颜色。
- **一切对齐栅格**：元素起始 / 结束位置落在 88px 边距与 8px 节奏上。
- **一个要点 = 一张正文卡**：N 个要点生成 N 张正文，不堆叠。
