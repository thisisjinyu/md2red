# 03 · 内容 → 卡片映射规则

把一篇 Markdown / 结构化文本自动拆成一套卡片。

## 映射表

| 输入（文本） | 输出（卡片元素） |
|---|---|
| 系列名 / 编号 | 页眉左侧系列标识 + 右侧页码/进度点 |
| `# 标题` | 封面主标题（Display） |
| 导读 / 首段摘要 | 封面副文（Lead） |
| `## 小标题` | 正文卡 H2 |
| 要点句 / 段落 | 正文卡 point |
| `**数字 + 单位**` | 数据块 num（数字）+ em（单位） |
| 数据说明句 | 数据块 cap |
| `> 出处` / 引用 | 数据块 src |
| 结尾 / 号召 | 尾页收尾句 + CTA |
| 全文要点集合 | 尾页 recap 列表 |

## 规则

1. **一个 `##` 要点 = 一张正文卡**。N 个要点 → N 张正文 + 1 封面 + 1 尾页。
2. 每张正文最多一个数据块；数据应含 **数字 + 单位 + 说明 + 出处**四件。
3. 标题控制在两行以内；要点控制在 2–3 行。
4. 数据出处必须真实；未核实前标「示例数据 · 待核」。
5. recap 列表逐条对应正文序号，文案压缩到一行。

## 示例（HE-0001）

```
# 改变人生的 6 个晨间仪式
导读：不靠意志力，靠系统。

## 醒来先喝 300ml 温水
一夜会流失约 0.5L 水分。
**10% ↓** 轻度脱水下认知下降一成。
> Journal of Nutrition
```
→ 1 封面 + 6 正文 + 1 尾页，如 [`prototype/`](../prototype/) 三套原型所示。
