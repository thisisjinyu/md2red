# 03 · 提取与语义判断（Extract → Classify → Route）

> 严格照 guizang `content-planning` 的方法。提取不是机械拆分，是按语义做内容规划，再判形态、挑变体、拼组合。
> （旧版「1 个 `##` = 1 张卡」的死模版逻辑已废弃。）

## A. Extract — 压缩阶梯（guizang Compression Ladder）

从原文逐层压缩：

1. **核心主张**：一句话。
2. **读者承诺**：刷完这套能得到什么。
3. **段落地图**：4-8 个意思。
4. **页面钩子**：短、具体、不抽象。
5. **正文碎片**：只留解释该视觉所需的词。

原文不整篇上图——图组是「视觉提纲」，勾人去看正文。

### 页数规则（guizang Page Count）

- 600-1000 字：5-7 张
- 1000-1800 字：7-9 张
- 超长：压到 8-10 张，其余进正文
- 多页下半区大片空 → 合并相邻意思

## B. 页面计划模板（guizang Rednote Page Plan，每套必填）

```text
标题：
受众：
风格：mint / acid / maillard
图片素材：

01 封面
- 钩子：
- 视觉：
- 底部看点：

02 正文页
- 要点：
- 文案：
- 视觉：
- 版式（变体）：
...

0N 尾页
```

## C. Classify — 判内容形态

按信号判（详见 04 路由表）：

- **编号清单体**：N 个/N 种，并列条目
- **步骤/流程体**：有先后顺序
- **单点强论体**：一个结论撑全篇
- **对比/测评体**：多项横向比

## D. Route — 挑变体 + 拼组合

- 形态 → 变体族（1:N），按「选择依据」挑：条目数 / 是否带后果 / 是否带数字 / 受众语气 / 是否对比。
- **Deck = 封面变体 + 正文变体（可混用）+ 尾页变体**，按语义拼，不写死。详见 `04-templates.md`。

## E. Good Page Roles（guizang，避免每页同款卡）

钩子封面 / 问题场景 / 误区 vs 真相 / 清单 / 对比 / 证据截图 / 一句大结论 / 步骤流程 / 清单选购 / 「记住这三点」尾页。

## F. Copy Rules（guizang）

- 标题平实锐利；少用「背景介绍」这类抽象标签。
- 用动词 + 后果：「高帮鞋会让脚更闷」优于「鞋子选择」。
- 健康类引导落到「身体感受 / 可执行」：更快、更省力、能落地。
- 去 AI 味：不堵「超绝 / 神器 / 必吃」，不软「一定要千万别」。

## G. Cover Hook 套路（guizang，挑一个）

「先看这 5 个点」/ 「我替你踩过坑了」/ 「新手别急着…」/ 「从此不用手动…」/ 「这个真的改变用法」。

避免：超长完整标题、空泛点作、一个封面塞多个主张。

## H. Captions / Metadata（guizang，套系一致）

- 分类行：`HEALTH / HABIT / 2026`
- 期号行：`晨间系统 / 第 01 期`
- 页码：`03 / 08`
- 紧凑标签：`DO` `DON'T` `WHY` `NOTE`

## I. 示例：同形态 → 不同变体

- **HE-0001《6 个晨间仪式》**：每条是可执行习惯、无重后果 → **L-CHECK 轻型** + 期号封面 + 「记住这三点」尾页。
- **WL-0001《越减越胖的 5 大原因》**：每条是原因/代价 → **L-LEDGER 重型** + 钩子封面（「我替你踩过坑了」）+ 小结账本尾页。

同为编号清单体，版式不同——这就是不固定死模版。
