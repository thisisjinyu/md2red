# 路由实跑 · HE-0001 与 WL-0001（Extract → Classify → Route → Fill → Render）

> 目的：同为「数字型标题 + 条目正文」，证明系统会按语义判出不同版式。两篇都走完整 03 流水线，最后落到不同变体。

## 总览对照

| | HE-0001 | WL-0001 |
|---|---|---|
| 标题 | 改变人生的 6 个晨间仪式 | 越减越胖的 5 大原因 |
| 形态（Classify） | 编号清单体 | 编号清单体 |
| 每条性质 | 可执行习惯 · 无重后果 | 原因/机制 · 带后果代价 |
| 受众语气 | 种草 · 轻松鼓励 | 反共识 · 警示 |
| 选中正文变体 | **L-CHECK 轻型** | **L-LEDGER 重型** |
| 封面 | 期号封面（轻） | 钩子封面「我替你踩过坑」 |
| 尾页 | 记住这三点 | 小结账本（先停掉这 5 件） |
| 渲染输出 | templates/L-CHECK.html | templates/L-LEDGER.html |

→ 同形态、同为编号清单，版式仍不同。这就是不固定死模版。

## A. HE-0001 实跑

### 1) Extract（压缩阶梯）
- 核心主张：早晨前 60 分钟用固定动作锁定，一天自动进正轨。
- 读者承诺：6 个能立刻照做的晨间动作。
- 段落地图（6）：温水 / 晨光 / 蛋白早餐 / 唤醒身体 / 离屏 / 写要事。
- 页面钩子：「不靠意志力，靠系统」。
- 正文碎片：每条 = 动作 + 一句为什么。

### 2) Classify
并列的 6 个「该做的事」，无先后强依赖、无严重后果 → 编号清单体。

### 3) Route（选择依据打分）
- 条目数 N=6 → 适合单屏清单（≤8）。
- 每条带后果/代价？否 → 不需要账本的后果列。
- 带数字？弱 → 不走 L-DATA。
- 有先后顺序？否 → 不走 T-STEP。
- 受众语气：种草/轻松 → 浅底、轻字重。
→ **L-CHECK 轻型**。封面用期号（系列感），尾页用「记住这三点」收束。

### 4) Fill（页面计划）
01 封面 期号+钩子；02 正文 L-CHECK 6 行；03 尾页 三点+CTA。

### 5) Render
templates/L-CHECK.html（3 帧）。

## B. WL-0001 实跑

### 1) Extract
- 核心主张：越减越胖，多半是 5 个反直觉的坑。
- 读者承诺：5 个让努力白费的原因 + 怎么躲。
- 段落地图（5）：极端节食 / 只看秤 / 睡不够 / 无糖零食 / 报复性进食。
- 页面钩子：「不是你不够自律」。
- 正文碎片：每条 = 原因 + 后果。

### 2) Classify
5 个并列「原因」，每条都带机制与代价 → 编号清单体。

### 3) Route
- 条目数 N=5 → 单屏。
- 每条带后果/代价？是 → 需要后果列 → 账本。
- 受众语气：反共识/警示 → 重字重、满宽行、序号大。
→ **L-LEDGER 重型**。封面用钩子「我替你踩过坑」，尾页用小结账本。

### 4) Fill
01 钩子封面；02 L-LEDGER 5 行（序号+原因+后果）；03 小结尾页。

### 5) Render
templates/L-LEDGER.html（3 帧）。

## 结论
路由的差异点全在 Classify 之后的「选择依据」：**是否带后果/代价** 与 **受众语气** 两条，把同一种「编号清单」分流到轻型 L-CHECK vs 重型 L-LEDGER。后续接入更多文章时，AI 按同一打分表判断，可混用不同封面/尾页变体组合。
